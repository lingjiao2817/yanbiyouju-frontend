# 《言必有据》准确性验证方案

## 1. 学长问的“准确性”到底是什么

这个项目不是做“AI 检测率”，而是做“AI 生成内容中的事实核查风险识别”。

所以我们验证的重点不是：

- 这段文本是不是 AI 写的

而是：

- 系统能不能识别出疑似虚构文献、疑似虚构 DOI、无来源统计数据、可疑法条等高风险内容

## 2. 本次验证目标

本次先做一个课程项目级别的小型验证，回答两个问题：

1. 有风险的文本，系统能不能识别出来
2. 没明显风险的文本，系统会不会误报太多

## 3. 验证样本构成

样本文件：

```text
backend/validation/test_cases.json
```

当前提供 20 条文本：

- 10 条高风险样本
- 10 条低风险样本

每条样本都包含：

- `id`
- `title`
- `expected_risk`
- `expected_categories`
- `text`

## 4. 评价指标

### 命中率

高风险样本中，被系统判定为 `medium` 或 `high` 的比例。

通俗理解：

- 本来有问题的文本，系统抓出来了多少

### 误报率

低风险样本中，被系统判定为 `medium` 或 `high` 的比例。

通俗理解：

- 本来问题不明显的文本，被系统误判了多少

### 类别命中情况

看系统是否识别出了目标风险类别，比如：

- 文献引用
- 法条法规
- 数据统计
- 人名机构
- 绝对化结论

## 5. 验证方法

### 方法一：直接跑自动脚本

```powershell
cd backend
python validation/evaluate_cases.py
```

如果 `python` 不可用：

```powershell
cd backend
py validation/evaluate_cases.py
```

脚本会：

1. 逐条读取 `test_cases.json`
2. 调用当前后端规则引擎
3. 输出每条样本的预测结果
4. 统计总样本数、命中率、误报率、类别命中情况

### 方法二：人工展示版

如果是答辩或课堂展示，可以挑 3 到 5 条样本手动输入前端页面演示：

1. 输入高风险文本
2. 展示系统标出的风险项
3. 解释为什么判定为高风险
4. 再输入低风险文本
5. 展示误报较少的结果

## 6. 结果怎么说

你可以这样解释：

“我们目前做的是小样本验证，不是大规模工业级评测。验证目标是看系统能否识别典型 AI 编造内容中的高风险事实信息。”

“我们把样本分成高风险和低风险两类，观察系统对高风险样本的命中情况，以及对低风险样本的误报情况。”

## 6.1 当前一次验证结果

本次已完成一轮小样本验证，使用 `backend/validation/test_cases.json` 中的 20 条样本进行测试：

- 高风险样本：10 条
- 低风险样本：10 条

自动评测结果如下：

- 命中率：90.00%
- 误报率：0.00%
- 类别命中率：86.67%

结果解读：

- 在 10 条高风险样本中，系统成功识别出 9 条
- 在 10 条低风险样本中，系统没有出现误报
- 在风险类别识别上，系统对文献、数据、法条、人名机构等风险类型具有较好的初步识别能力

当前可以得出的结论：

- 当前规则引擎对典型 AI 编造内容已经具备较好的初步识别效果
- 系统在这批小样本上的误报控制较好
- 目前结果适合用于课程项目展示与阶段性汇报

当前结果的边界：

- 这是一轮课程项目级别的小样本验证，不代表大规模真实场景下的最终性能
- 后续仍需要继续扩充真实样本、补充人工标注，并在接入 Crossref 与大模型能力后继续评估

## 7. 答辩时建议说法

### 简洁版

“我们已经做了一个小型准确性验证。不是检测文本是不是 AI 写的，而是验证系统对高风险事实错误的识别能力。我们准备了高风险样本和低风险样本，统计命中率和误报率，用来评估当前规则引擎的效果。”

### 稍正式版

“本系统的评估对象不是 AI 文本识别，而是 AI 生成文本中的事实核查风险识别。我们构造了包含疑似虚构文献、可疑 DOI、无来源统计数据和可疑法条的高风险样本，以及若干低风险规范样本，通过命中率、误报率和风险类别识别情况对系统进行了初步验证。”

## 8. 目前验证的边界

这次验证主要说明：

- 当前规则引擎是否有基本可用性
- JSON 规则库是否能有效增强风险命中

这次验证还不能完全说明：

- 跨学科泛化能力
- 大规模真实数据上的稳定性
- 接入 Crossref 和大模型之后的最终准确率

## 9. 下一步适合继续补的内容

- 增加更多真实论文样本
- 增加人工标注字段，比如“应命中的具体句子”
- 增加 precision / recall / F1
- 接入 Crossref 后做对比实验
- 接入大模型语义核查后做 ablation 对比
